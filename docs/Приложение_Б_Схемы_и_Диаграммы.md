# ПРИЛОЖЕНИЕ Б. СТРУКТУРНЫЕ СХЕМЫ И ДИАГРАММЫ СИСТЕМЫ

В данном приложении приведены графические схемы архитектуры, логической структуры базы данных и основных алгоритмов функционирования игровой платформы **CYBER_OPS://CORE**. Диаграммы описаны на языке разметки схем **Mermaid.js**.

---

## Б.1. Логическая схема базы данных (Entity-Relationship Diagram)

Реляционная база данных системы функционирует под управлением СУБД SQLite. Архитектура базы состоит из центральной сущности `agents` (аккаунты пользователей) и связанных с ней дочерних таблиц по внешним ключам (Foreign Keys) с каскадными ограничениями целостности.

```mermaid
erDiagram
    agents ||--o{ unlocked_skills : "имеет"
    agents ||--o{ xp_log : "получает"
    agents ||--o{ inventory : "владеет"
    agents ||--o{ chat_messages : "отправляет"

    agents {
        int id PK "AUTOINCREMENT"
        string username UK "NOT NULL, 3-16 симв."
        string password_hash "NOT NULL, bcrypt"
        string alignment "DEFAULT 'UNASSIGNED'"
        int level "DEFAULT 1"
        int xp "DEFAULT 0"
        int next_level_xp "DEFAULT 1000"
        string title "DEFAULT 'RECRUIT'"
        int game_credits "DEFAULT 500"
        int premium_credits "DEFAULT 0"
        int skill_points "DEFAULT 3"
        int pvp_enabled "DEFAULT 0 (Boolean)"
        int pvp_elo "DEFAULT 1000"
        int pvp_wins "DEFAULT 0"
        int pvp_losses "DEFAULT 0"
        int pvp_streak "DEFAULT 0"
        datetime created_at "DEFAULT CURRENT_TIMESTAMP"
    }

    unlocked_skills {
        int id PK "AUTOINCREMENT"
        int agent_id FK "REFERENCES agents(id)"
        string skill_id "NOT NULL"
        datetime unlocked_at "DEFAULT CURRENT_TIMESTAMP"
    }

    xp_log {
        int id PK "AUTOINCREMENT"
        int agent_id FK "REFERENCES agents(id)"
        string source "NOT NULL (mission/book/pvp)"
        string label "NOT NULL"
        int xp_amount "NOT NULL"
        datetime created_at "DEFAULT CURRENT_TIMESTAMP"
    }

    inventory {
        int id PK "AUTOINCREMENT"
        int agent_id FK "REFERENCES agents(id)"
        string item_name "NOT NULL"
        string item_type "NOT NULL (cpu/firewall/neurolink/network_adapter)"
        string rarity "DEFAULT 'common'"
        string buff "TEXT"
        int equipped "DEFAULT 0 (Boolean)"
    }

    chat_messages {
        int id PK "AUTOINCREMENT"
        int agent_id FK "REFERENCES agents(id)"
        string channel "DEFAULT 'global'"
        string message "NOT NULL"
        datetime created_at "DEFAULT CURRENT_TIMESTAMP"
    }
```

## Б.2. Схема архитектуры системы (Component & Communications Diagram)

Платформа представляет собой клиент-серверное одностраничное веб-приложение (SPA). Защита обмена данными реализована с помощью токенов доступа JWT (JSON Web Token), которые отправляются в HTTP-заголовках Authorization при каждом запросе к защищенным эндпоинтам.

```mermaid
graph TD
    %% Стилизация узлов
    classDef client fill:#0d2030,stroke:#00e5ff,stroke-width:2px,color:#fff;
    classDef api fill:#1a0f30,stroke:#d946ef,stroke-width:2px,color:#fff;
    classDef db fill:#0f1f15,stroke:#22c55e,stroke-width:2px,color:#fff;
    classDef external fill:#1a1a1a,stroke:#555,stroke-width:2px,color:#888;

    %% Клиентская часть
    subgraph Frontend_Client ["Клиентское веб-приложение (React SPA / Vite)"]
        ui[Интерфейс Пользователя / UI]
        router[React Router 7]
        store[Zustand Global Store]
        storage[(LocalStorage JWT)]
    end
    class ui,router,store,storage client;

    %% Серверная часть
    subgraph Backend_Mainframe ["Серверный интерфейс (Node.js & Express.js)"]
        api_gateway{API Gateway}
        auth_mw{authMiddleware}
        router_api[Express API Routes]
    end
    class api_gateway,auth_mw,router_api api;

    %% База данных
    subgraph Database_Layer ["Хранилище Данных"]
        sqlite_db[(cyber_ops.db SQLite)]
    end
    class sqlite_db db;

    %% Внешние связи
    ui -->|1. Событие UI| store
    store -->|2. Чтение JWT| storage
    store -->|3. HTTP API Request + Bearer JWT| api_gateway
    api_gateway -->|4. Валидация подписи JWT| auth_mw
    auth_mw -->|Успешно: Добавление agentId к req| router_api
    auth_mw -.->|Ошибка 401: Invalid Token| store
    router_api -->|5. SQL Запрос| sqlite_db
    sqlite_db -->|6. SQL Данные| router_api
    router_api -->|7. HTTP JSON Response| store
    store -->|8. Обновление стейта| ui
    router -->|9. Переключение страниц| ui
```

## Б.3. Блок-схема игровых механик (Player Progression & XP Logic Flow)

Ниже представлена детальная блок-схема процесса начисления опыта (XP), прогрессивного масштабирования необходимого опыта для следующего уровня (+15% с каждым уровнем) и начисления очков навыков (+2 SP за каждый Level-up).

```mermaid
flowchart TD
    classDef process fill:#112233,stroke:#00e5ff,stroke-width:2px,color:#fff;
    classDef decision fill:#221133,stroke:#d946ef,stroke-width:2px,color:#fff;
    classDef success fill:#113322,stroke:#22c55e,stroke-width:2px,color:#fff;
    classDef start_end fill:#331111,stroke:#ef4444,stroke-width:2px,color:#fff;

    Start([1. Завершение задания Агентом]):::start_end --> AddXP[2. Отправка POST /api/xp/add]:::process
    AddXP --> FetchDB[3. Запрос текущих параметров: xp, level, next_level_xp, skill_points]:::process
    FetchDB --> CalculateXP[4. Новое значение: xp = xp + amount]:::process
    
    LevelCheck{5. Проверка: xp >= next_level_xp ?}:::decision
    LevelCheck -- Да --> IncrementLevel[6. Повышение уровня: level = level + 1]:::process
    IncrementLevel --> SubtractXP[7. Вычитание остатка: xp = xp - next_level_xp]:::process
    SubtractXP --> AddSP[8. Начисление очков навыков: skill_points = skill_points + 2]:::process
    AddSP --> ScaleXP[9. Рост сложности: next_level_xp = next_level_xp * 1.15]:::process
    ScaleXP --> LevelCheck
    
    LevelCheck -- Нет --> UpdateAgent[10. Определение нового звания Title по уровню]:::process
    UpdateAgent --> SaveDB[11. UPDATE Таблицы agents в БД]:::process
    SaveDB --> LogXP[12. INSERT Записи в xp_log для истории]:::process
    LogXP --> SyncClient[13. Синхронизация состояния Zustand Store и UI]:::success
    SyncClient --> End([14. Обновленный интерфейс Dashboard]):::start_end
```
